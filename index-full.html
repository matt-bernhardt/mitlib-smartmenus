SmartMenus for MIT Libraries

This page is still in progress

This page should place the menu within the navbar context, and load the needed global styles in place on libraries.mit.edu.

Interaction with the menu should be supported in touch interfaces, keyboard navigation, screenreaders, and mouse interactions. All of them should be possible. The spacebar will open a highlighted menu, while escape will close them.

Issues

This page has the following known issues:
  • Between 568 and 768 pixels in width, the menu stacks and breaks theming
  • Below 568 pixels in width, the meatball appears, but no javascript listens for its invocation
  • Visual indicator of a submenu (triangle) is probably off-brand - alternative has been identified, but not implemented
  • "About" and "Ask us" flyouts are extending outside the content column
  • "Ask us" flyout is too wide (min/max width is wonky)
  • SVG icons for "Ask us" and "Account" items are missing
  • Flyout elements are unstyled
  • Flyout links don't have subtext
  • LibChat widget isn't implemented
  • Probably other branding problems

Feedback?

Please contact Matt Bernhardt with any feedback about this work.
